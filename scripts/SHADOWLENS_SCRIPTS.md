# ShadowLens Scripts

Deployment and testing scripts for the ShadowLens dApp.

## Scripts Overview

### `deploy-shadowlens.ts`
**Deploy the ShadowLens contract to Ritual Chain**

```bash
npx hardhat run scripts/deploy-shadowlens.ts --network ritual
```

**What it does:**
- Deploys ShadowLensAgentConsumer contract
- Saves deployment info to `examples/shadowlens-frontend/deployment.json`
- Creates `.env.local` with contract address in frontend directory
- Displays all deployment details and next steps

**Output:**
- Contract address
- Owner information
- Precompile addresses (Sovereign Agent, HTTP Precompile, AsyncDelivery)
- Deployment timestamp

### `setup-oracle.ts`
**Authorize the oracle to fulfill analysis requests**

```bash
npx hardhat run scripts/setup-oracle.ts --network ritual
```

**What it does:**
- Reads contract address from `deployment.json`
- Authorizes deployer account as oracle
- Enables calling `fulfillAnalysis()` function

**Prerequisites:**
- Must run `deploy-shadowlens.ts` first
- Must be called by contract owner

### `test-shadowlens.ts`
**Test the deployed contract**

```bash
npx hardhat run scripts/test-shadowlens.ts --network ritual
```

**What it tests:**
1. Owner verification
2. Analysis request creation
3. Oracle authorization
4. Analysis fulfillment
5. Precompile address configuration

**Note:** Some tests may fail on local networks if precompiles aren't available (this is expected).

## Deployment Workflow

```
1. Deploy Contract
   → deploy-shadowlens.ts
   → Creates deployment.json
   → Generates .env.local for frontend

2. Authorize Oracle
   → setup-oracle.ts
   → Enables backend fulfillment

3. Test Deployment
   → test-shadowlens.ts
   → Verifies contract functionality

4. Configure & Start
   → Frontend: npm run dev
   → Backend: npx ts-node listener.ts
```

## Environment Setup

### Required Environment Variables

```bash
# For deployment script
PRIVATE_KEY=your_private_key_here
RITUAL_RPC_URL=https://rpc.ritualfoundation.org

# For frontend (auto-generated by deploy script)
NEXT_PUBLIC_SHADOWLENS_ADDRESS=0x...
NEXT_PUBLIC_WC_PROJECT_ID=your_wc_project_id
```

### Network Configuration

Add to `hardhat.config.ts`:

```typescript
networks: {
  ritual: {
    url: process.env.RITUAL_RPC_URL || "https://rpc.ritualfoundation.org",
    accounts: process.env.PRIVATE_KEY ? [process.env.PRIVATE_KEY] : [],
    chainId: 1979,
  },
}
```

## Troubleshooting

### Script hangs or times out
- Check RPC endpoint is accessible
- Verify network connectivity
- Ensure account has sufficient balance

### "Unknown requestId" error in test
- This is expected - the test creates a mock request ID
- The contract logic is still validated

### Precompile calls fail
- This is expected on non-Ritual networks
- Precompiles only work on Ritual Chain
- Local tests focus on contract structure

### Oracle authorization fails
- Ensure you're using the same account that deployed the contract
- Owner can authorize other accounts with `setOracle()`

## Tips

1. **Save Deployment Info**: The `deployment.json` file is created automatically and contains all important addresses.

2. **Multiple Deployments**: Each new deployment creates a new contract instance. Update `.env.local` and backend configs accordingly.

3. **Oracle Authorization**: You can authorize multiple accounts:
   ```typescript
   await shadowLens.setOracle(backendAddress, true)
   await shadowLens.setOracle(anotherAddress, true)
   ```

4. **Gas Optimization**: Deployment typically uses ~500k-1M gas. Ensure sufficient balance.
